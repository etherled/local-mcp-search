### 4. 检索与索引能力

- `code_exact_search`
  - 基于 `ripgrep`
  - 适合查符号名、错误文本、配置键、路径、标识符

- `symbol_search`
  - 基于声明模式匹配
  - 用于定位函数、类、接口、类型、常量的定义位置

- `code_semantic_search`
  - 基于本地 embedding + `LanceDB`
  - 用于查相似逻辑、模式、相关实现

- `kb_search`
  - 基于知识文档切块后的语义检索

- `open_spans`
  - 根据路径和行号回取精确上下文
  - 用于控制上下文体积，避免一次展开整文件

### 5. 向量存储升级

- 语义索引底座从简单 JSON 存储升级为 `LanceDB`
- 索引目录为 `.mcp-index/lancedb`
- 元数据为 `.mcp-index/metadata.json`

### 6. 重建与更新机制

- 支持三种 `reindex` 模式：
  - `auto`
  - `full`
  - `incremental`

- 在 Git 仓库中：
  - 记录 `last_indexed_commit`
  - 结合 `git diff`、未暂存变更、已暂存变更、未跟踪文件判断受影响路径

- 在非 Git 目录中：
